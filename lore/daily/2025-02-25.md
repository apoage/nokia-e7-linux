## Key Findings

### ARM Cortex-A8 + NEON
- 2-cycle writeback hazard is the #1 optimization target (25-40% gain)
- Dual-issue ARM+NEON makes PLD/address math free in NEON shadow
- VQMOVUN gives free clamping — saves 2 instructions per pixel output
- VMULL.P8 polynomial multiply for ~1 cycle/8-byte tile hashing
- 64KB on-chip SRAM at 0x40200000 as software scratchpad
- ACTLR bit 11 (NEON double-width store) may not be enabled by default

### C64x+ DSP (IVA2.2)
- **iMX accelerator discovered** — hidden image coprocessor with hardware SAD,
  DCT/IDCT, block matching. Runs concurrent with DSP core.
- EDMA3 can deinterleave Bayer patterns in hardware (zero DSP cycles)
- EDMA3 chaining creates fully autonomous double-buffered pipelines
- L1D bank conflicts are the DSP perf killer — pad strides to avoid
- L2 SRAM shared with ARM at 0x5C800000 = zero-coherency shared buffer
- IVA2 can hit 800 MHz with ABB Forward Body Bias

### OMAP3 ISP
- Preview engine CFA filter has 192 programmable coefficients
- H3A AF engine with custom IIR coefficients = hardware edge detection map
- Histogram module has 4 independent ROI regions
- Resizer works standalone (memory-to-memory) for hardware image pyramids
- A-law compression in CCDC for 37% burst capture bandwidth savings
- All submodules can operate standalone from memory

### PowerVR SGX530/540
- Existing SGX540 RE work integrates well (same ISA family)
- USSE ISA partially decoded: 11 register banks, instruction encoding known
- 10-cycle WAW hazard confirmed experimentally
- PDS (Programmable Data Sequencer) bootstraps microkernel
- Microkernel has 0xDEADBEEF patch points filled at load time

### System-Level
- SDMA constant fill = hardware memset, transparent copy = chroma-key blit
- DSS VID overlays do free YUV→RGB, 5-tap scaling, alpha compositing
- No hardware cache coherency ARM↔DSP — use L2 SRAM or explicit flush
- SmartReflex Class 1.5 preferred on 3630
- ABB mandatory for >800 MHz ARM / >660 MHz DSP
