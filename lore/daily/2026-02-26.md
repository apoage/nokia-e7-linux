# Day 2 — 2026-02-26 — Firmware Cracked Open

## Summary
Extracted all three ROFS filesystems from Nokia E7 stock firmware (Euro3-Czech
variant, RM-626 v111.040.1511). Major hardware discovery: the E7 has a
**Broadcom BCM2727 VideoCore co-processor** in addition to the OMAP3630 SoC.

## QEMU Status
- QEMU 10.x on this system has **no OMAP3/beagleboard machine type** — removed
  from mainline. Only OMAP machine is `sx1` (ancient OMAP310).
- Available Cortex-A8 target: `realview-pb-a8` (no OMAP peripherals)
- Available generic: `virt` machine (paravirtualized)
- **Register remapping claim DEBUNKED**: Standard OMAP3 L4 addresses (0x48xxxxxx)
  confirmed intact in firmware. The real barrier is BB5 security, not remapping.
- Path forward: build custom QEMU OMAP3630 machine, or build qemu-linaro from source.
- See: `docs/qemu-machine-design.md` for full analysis.

## Firmware Extraction

### Tools
- **saur0n/unpacker** — built from source with local libunix++ dependency
  (symlink trick: `ln -s libunixpp-master unpacker-master/unix++`)
- Build command: `make CFLAGS="-Wall -Wno-unused -I." LIBRARIES="-lstdc++
  -L.../libunixpp-master -lunix++ -lcrypto -lz"`
- Run with: `LD_LIBRARY_PATH=.../libunixpp-master ./unpacker -o <outdir> <file>`

### FPSX Format Notes
- Header: signature byte (0xB2) + 4-byte header size + TLV properties
- Block types: 0x17 (binary), 0x27 (ROFS hash), 0x28 (cert), 0x2E (user area),
  0x30 (eMMC — **unsupported** by unpacker), 0x3A, 0x49
- The unpacker extracts blocks to `rofs.img` at their OneNAND logical offsets

### ROFS Extraction
- The `rofs.img` is a flat flash image with partition table at offset ~0x140300
- **BB5 header magic**: `0x809795A3` (little-endian)
- **ROFS magic**: `0x53464F52` ("ROFS" in LE)
- The BB5 header in E7 firmware uses **4096-byte padding** (not 1024 as unpacker
  assumes). ROFS filesystem starts at BB5_offset + 0x1000.
- Workaround: extract pure ROFS with `python3 -c "skip 0x1000 bytes"`

## Register Map Analysis
Searched the pre-ROFS kernel area (0x80000-0x1C0000) for OMAP3 register addresses.

**Standard OMAP3 addresses found and CORRECT:**
- 0x48004000 SCM (System Control Module)
- 0x4809C000 MMC/SD1
- 0x48306000 PRCM_IVA2
- 0x49020000 UART3
- 0x49032000 GPTIMER2
- 0x49042000 UART4

**The "register remapping" claim from postmarketOS may be overstated.** Standard L4
peripheral addresses appear intact in the firmware. The actual barrier may be more
about BB5 security/signing than register remapping.

**909 unique addresses in 0x50xxxxxx (GPMC range)** — likely NAND controller
and possibly BCM2727 external interface.

### BCM2727 VideoCore III Details
- Connected to OMAP3630 via **CCP2** (Compact Camera Port 2 / MIPI CSI-1)
- Uses **VCHI** (VideoCore Host Interface) messaging protocol over CCP2
- Driver: `DVCDriver` — loads VideoCore firmware image at boot
- Capabilities confirmed from VLL files:
  - Video encode: H.264, H.263, MPEG-4
  - Video decode: VP6, VC-HW decode
  - JPEG hardware encode
  - Camera ISP tuning (separate from OMAP3 ISP)
  - Image processing: denoise, stabilize, depurple, face detect (ArcSoft)
  - HDMI output
- **Same architecture family as Raspberry Pi GPU** — VideoCore documentation
  from Broadcom's open-source release may apply

### Atmel Touch Controller
- Full driver stack: sensor, boot flash, IRQ handler, charger detection
- The charger detection via the touch controller is Nokia-specific (common in
  capacitive touch designs — the same ADC detects both touch and USB charger)

## QEMU OMAP3630 Machine — Implemented

Built a custom QEMU OMAP3630 machine definition from scratch (QEMU 10.x).
Pattern: allwinner-a10/cubieboard SoC+board split.

### Files Created
| File | Purpose | Lines |
|------|---------|-------|
| `hw/arm/omap3630.c` | SoC wiring (CPU, INTC, timers, UART, stubs) | ~280 |
| `hw/arm/nokia_e7.c` | Board def (256MB RAM, machine registration) | ~60 |
| `hw/intc/omap3_intc.c` | OMAP3 96-line interrupt controller | ~500 |
| `hw/timer/omap3_gptimer.c` | GP timer with overflow/match/capture IRQ | ~300 |
| `include/hw/arm/omap3630.h` | SoC state struct + address map defines | ~80 |
| `include/hw/intc/omap3_intc.h` | INTC header | ~30 |
| `include/hw/timer/omap3_gptimer.h` | GPTimer header | ~30 |
| `hw/{arm,intc,timer}/Kconfig` | Build config entries (modified) | — |
| `hw/{arm,intc,timer}/meson.build` | Build system entries (modified) | — |

### Emulated Peripherals
- **CPU**: Cortex-A8 with MIDR 0x413fc082 (r3p2, avoids ES1.0 path)
- **INTC**: Full 96-line controller with SIR protocol, priority sorting
- **UART3**: serial_mm at 0x49020000, reg-shift=2, IRQ 74
- **GPT1**: 32.768 kHz timer at 0x48318000, IRQ 37
- **GPT2**: 38.4 MHz timer at 0x49032000, IRQ 38
- **32K Sync Timer**: Free-running counter at 0x48320000
- **SRAM**: 64KB at 0x40200000
- **TAP**: IDCODE 0x2B89102F (OMAP3630 ES1.2)
- **CM/PRM stubs**: Return 0xFFFFFFFF (DPLLs locked, modules functional)
- **SCM stub**: Returns 0 (correct feature detection)
- **Unimplemented stubs**: GPIO1-6, I2C1-3, SMS, SDRC, GPMC, HSMMC1, DSS, MUSB, WDT2

### Boot Test Results
1. **Bare-metal UART test**: PASS — "Hello from OMAP3630!"
2. **Linux 6.12 earlycon**: PASS — identifies as OMAP3630/DM3730 ES1.2
3. **INTC**: Working (96 IRQs detected, prioritized)
4. **32K clocksource**: Registered, sched_clock running
5. **DPLLs**: All report locked (CM all-ones stub)
6. **STALL**: After "Console: colour dummy device" — GPT1 init fails (-ENODEV)

### Root Cause of Stall
The OMAP hwmod (hardware module) subsystem tries to enable GPT1 by:
1. Writing CLKEN bits in CM_FCLKEN_WKUP / CM_ICLKEN_WKUP
2. Polling CM_IDLEST_WKUP for module idle status transition
3. The simplistic all-ones CM stub always returns "idle=functional" but the
   hwmod state machine expects specific transitions from the enable sequence
4. Without a working tick timer, the scheduler can't run → boot hangs

### DTS Solution
Minimal hand-written DTS failed (clock tree crashes). Switched to using
mainline `omap36xx.dtsi` include — placed DTS in kernel's `arch/arm/boot/dts/ti/omap/`
directory and built with kernel's DTS toolchain. Full clock tree now registers.

### Key Learnings (QEMU 10.x)
- `const Property[]` required (no DEFINE_PROP_END_OF_LIST)
- `#include "hw/qdev-properties.h"` needed for Property type
- `#include "exec/address-spaces.h"` for get_system_memory()
- SysBusDevice children MUST use sysbus_realize(), not qdev_realize()
- Setting cpu.midr before qdev_realize() overrides the default MIDR
