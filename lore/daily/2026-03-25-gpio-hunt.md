# 2026-03-25: The GPIO Hunt

## Goal
Read GPFSEL registers (28 bytes at physical 0x20200000) from the running phone.
These contain the complete pin mux configuration set by GENIO_INIT at boot.

## BCM2708 GPIO Header Found
- Web research agent found `gpio.h` from shacharr/videocoreiv-qpu-driver on GitHub
- BCM2708 (= BCM2835 without ARM) GPIO registers confirmed identical layout
- 7 GPFSEL registers covering 70 GPIO pins (BCM2835 only exposes 54)
- GP_SEN (Schmitt enable) at 0xA4 -- undocumented in BCM2835 datasheet
- GP_AJBCONF (JTAG bridge) at 0xC0 -- also undocumented
- Saved as docs/bcm2708-gpio.h (735 lines) + docs/bcm2708-hardware.h

## GPIO Driver Disassembly
- GPIO driver code at ROM 0xC0C00-0xC1900
- Found constant pool entries: 0xC8004150, 0xC8004428, 0xC8006028
- These are kernel virtual addresses for 3 GPIO bank bases
- Driver uses indirect addressing through C++ class member pointers
- Register layout differs from BCM2835 at detail level (byte-granularity offsets)
- But base address 0x20200000 confirmed by 300+ references in ROM

## Phone-Side Probing (gpfsel_probe.py v4)
Full 10-probe scan ran through probes 1-6 successfully:

### Discoveries from probe output
- **misty module**: kill_thread, get/set_process_priority, terminate_thread
- **envy module**: ECapabilityReadDeviceData, is_app_ready, set_app_ready
- **fshell_hal.exe**: HAL value dumper (2483 bytes, on C:\sys\bin)
- **memaccess.exe**: standalone memory access tool (6376 bytes!)
- **1024 files** total in C:\sys\bin
- **AF_INET = 2048** (Symbian uses different socket constant values)
- **location.gsm_location** and **positioning** modules available

### Probe 7 Crash
- FShell readmem triggered kernel panic + reboot
- Same TCB issue -- memoryAccess LDD DoCreate panics without TCB
- Log saved up to crash point (save() after each section worked)

## New Attack Vectors Identified
1. **fshell_hal.exe** -- may dump HAL values including HW info
2. **memaccess.exe** -- 6KB standalone tool, might bypass TCB issue
3. **misty module** -- process/thread control from Python
4. **envy module** -- capability checking/management
