## Bugs Found and Fixed

### 1. osc_sys_ck Rate = 0 (Crystal 0.0 MHz)
**Root cause**: `osc_sys_ck` is a `ti,mux-clock` under `prm_clocks` (child of
`prm@48306000`), NOT under the CM. It reads PRM offset 0xD40, not CM offset
0xD40. The PRM handler was returning 0xFFFFFFFF, giving mux index 7 — out of
bounds for 6 parents, resulting in rate 0.

**Fix**: PRM handler returns 3 at offset 0xD40 → selects 26MHz oscillator.
Similarly, `sys_ck` divider at PRM offset 0x1270 returns `(1u << 6)` for
divider = 1.

### 2. DPLL Zero Divisors (dpll2_fck, others)
**Root cause**: IVA2 CM_CLKSEL1_PLL was missing the DPLL2 divider bits
[21:19]. Without them, the divider was 0 → division by zero in clock rate
calculation.

**Fix**: Added `(1u << 19)` to IVA2 CLKSEL1_PLL return value (and ensured
all 5 DPLLs have correct M/N/M2 values).

### 3. I2C Infinite Spin (3.7 Million Reads)
**Root cause**: I2C_SYSSTATUS at offset 0x10, bit 0 (RDONE) was returning 0
from `create_unimplemented_device` (returns 0). PM init resets I2C controllers
and polls RDONE forever.

**Fix**: Created I2C stub handler returning SYSSTATUS.RDONE=1 at offset 0x10,
and STAT.NACK=1 at offset 0x08 (makes TWL PMIC probe fail fast instead of
spinning).

### 4. DPLL IDLEST Spin
**Root cause**: PM init puts DPLLs into bypass/stop mode by writing CM_CLKEN_PLL,
then polls CM_IDLEST waiting for DPLL status bit = 0 (not locked). But the CM
handler always returned 0xFFFFFFFF (all locked), regardless of what was written.

**Fix**: Added DPLL state tracking — four static variables track writes to
CM_CLKEN_PLL registers. The CM read handler derives IDLEST from tracked state
using a `dpll_is_locked()` helper.

### 5. PM Late Init Hang — The Critical SCM Fix
**Root cause**: `__omap2_common_pm_late_init` → `omap3_pm_init()` checks
`omap_type()`, which reads SCM register CONTROL_STATUS at offset 0x2F0.
Bits [10:8] = DEVICETYPE (0=test, 1=EMU, 2=HS, 3=GP). Our SCM stub returned
0 → DEVICETYPE = 0 (not GP). This caused the code to enter:
```c
if (omap_type() != OMAP2_DEVICE_TYPE_GP) {
    local_irq_disable();
    omap3_save_secure_ram_context();  // SMC instruction → HANG
    local_irq_enable();              // never reached
}
```
With IRQs disabled and no TrustZone monitor, the SMC caused an unrecoverable
hang with no console output.

**Fix**: SCM handler returns 0x300 at offset 0x2F0 (DEVICETYPE = GP = 3).
One line of code, hours of debugging.

### 6. MMC Deferred Probe Infinite Retries
**Root cause**: The OMAP HSMMC driver requests DMA channels via
`dma_request_chan()`. Without an sDMA controller emulated, this returns
`-EPROBE_DEFER` endlessly. Three MMC controllers × ~1s retry interval =
system stuck cycling through retries forever.

**Fix**: Disabled mmc1/2/3 and bandgap nodes in DTS with
`status = "disabled"`. The bandgap sensor had similar timeout issues
(eocz timed out waiting high).

## Debugging Techniques Used

### Finding the Hanging Initcall
- Added `ignore_loglevel` to kernel command line — reveals `initcall_debug`
  function names (which print at KERN_DEBUG level, normally suppressed)
- Output clearly showed: `calling __omap2_common_pm_late_init+0x0/0x68 @ 1`
  with no subsequent "returned" line

### QEMU `-d unimp` Analysis
- Showed 270K lines of unimplemented device reads
- 9 devices × 30,000 reads at offset 0x014 (SYSSTATUS) each — hwmod
  softreset bounded timeouts (10ms each)
- After all SYSSTATUS polls completed, unimp log stopped growing →
  confirmed the hang was inside a custom handler, not an unimplemented device

### Spin Detection
- Added repeat-read counters to CM/PRM/I2C/HSMMC handlers
- Useful for finding the I2C 3.7M spin and DPLL IDLEST spin
- Limitation: only detects consecutive reads to the SAME offset. Alternating
  patterns (read A, read B, read A, ...) slip through

### CM Write Tracing
- Logged CM writes #200-300 to stderr to understand the PM init register
  access pattern
- Showed DPLL mode toggling, clock domain configuration, DSS clock setup

## Architecture Insights

### OMAP3 Clock Tree (from PRM, not CM!)
Key lesson: the oscillator selection (`osc_sys_ck`) and system clock divider
(`sys_ck`) are under `prm_clocks` in the DTS, meaning they read PRM registers
(base 0x48306000), NOT CM registers (base 0x48004000). The PRM and CM are
separate register spaces despite both being "clock-related".

### OMAP3 Device Type Detection
`omap_type()` → SCM `CONTROL_STATUS` bits [10:8]:
- 0 = Test, 1 = EMU, 2 = HS (High Security), 3 = GP (General Purpose)
- Controls whether PM init runs secure-mode code (SMC instructions)
- Real Nokia E7 uses HS device type (BB5 security), but for emulation
  we must use GP to avoid TrustZone requirements

### Hwmod SYSSTATUS Polling
OMAP hwmod softreset sequence: write SYSCONFIG → poll SYSSTATUS offset 0x014,
bit 0 (RESETDONE). Timeout: ~10ms = ~30,000 reads at emulation speed. Every
`create_unimplemented_device` returns 0, so bit 0 is never set, causing
consistent 10ms timeouts per device. This is bounded and non-blocking.

## Task 010: Build Minimal Rootfs (DONE)

### Approach
BusyBox 1.36.1 initramfs (Option A from the task spec). Static ARM binary,
packed as gzip-compressed newc cpio, loaded via QEMU `-initrd`.

### Build Details
- BusyBox 1.36.1, statically linked for ARM (1.6MB binary, ~400 applets)
- Disabled `tc` applet — CBQ qdisc structs removed from kernel headers, breaks
  with GCC 15. BusyBox 1.37.0 also has sha-NI build failures with GCC 15.
- Init script: mounts proc/sysfs/devtmpfs, sets hostname, drops to `/bin/sh`
- Final initramfs: 1.1MB (`emulation/rootfs/initramfs.cpio.gz`)
- Reproducible build script: `emulation/rootfs/build-initramfs.sh`

### Bug: CRLF Line Endings in Init Script
The init script was written with CRLF line terminators. The kernel exec'd
`/init`, parsed the shebang as `#!/bin/sh\r`, tried to find `/bin/sh\r` →
ENOENT (error -2). The kernel then fell through to `/sbin/init` (BusyBox
default init) which partially worked but wasn't our custom script.

**Fix**: `sed -i 's/\r$//'` to convert to LF. Lesson: always verify with
`file` command that scripts have correct line endings before packing into cpio.

## Hardware-Software Symbiosis Plan

Created `lore/hw-sw-symbiosis.md` — comprehensive document selecting modern
software components optimised for the E7's 6 compute units, 256MB RAM, and
dual-storage architecture. Key decisions:

- **Compute**: Cortex-A8+NEON as primary workhorse, OMAP3 HW crypto for
  AES/SHA offload, BCM2727 VideoCore III as highest-value future target.
  Skip C64x+ DSP (dead since 2014) and PowerVR SGX530 (no open driver).
- **Filesystems**: EROFS+LZ4 for read-only rootfs (4KB-aligned, fast
  decompression), F2FS for writable eMMC partitions (flash-aware, low write
  amplification), UBIFS on UBI for OneNAND recovery.
- **Memory**: zram swap with LZ4 (192MB device, 2-3:1 compression ratio,
  zero flash wear). Budget: ~217MB for apps after kernel+FS overhead.
- **Userspace**: musl + BusyBox + dinit (dependency-based init). iwd over
  wpa_supplicant. cage (Wayland kiosk) on omapdrm with pixman+NEON.
- **Libraries**: libjpeg-turbo, zlib-ng, pixman, libsodium — all NEON-optimised.
- **Compiler flags**: `-march=armv7-a -mfpu=neon -mfloat-abi=hard -O2`

## Ideas: External Module Backpacking

The Rapuyama modem is a 3G-only chip (HSDPA 10.2 / HSUPA 2 Mbps, ARM11 @
680 MHz, 65nm). No path to 4G/LTE — it's a hard silicon limit (WCDMA RF
chain, no OFDMA support). With 3G sunsetting globally, an external radio
is the only long-term connectivity option.

**Concept**: Backpack an external 4G/5G module connected over USB OTG,
paired with its own battery (the BP-4L 1200mAh can't power a modern radio).
The E7 becomes a smart terminal; the external module handles connectivity.

Options explored (all equivalent — external radio + external battery):
- **USB LTE dongle + powerbank** — simplest, commodity hardware, WiFi or USB networking
- **SDR board (LimeSDR/PlutoSDR)** — fully open radio stack (srsRAN/GNURadio), most power hungry
- **SBC + mini PCIe modem** (e.g. Pi Zero 2W + Quectel EC25/RM502Q) — custom "modem backpack"

Constraints: no internal space in E7 casing (slider mechanism + AMOLED + battery fill it).
OMAP3 USB 2.0 (480 Mbps) is sufficient for LTE, bottleneck for 5G but usable.

Rapuyama could still serve as legacy 2G/3G voice/SMS radio alongside the external data modem.

## Task 011: QEMU Block Storage (DONE)

### Approach
virtio-mmio paravirtualised block device (Option A). One line of QEMU code,
one DTS node, two kernel config options. No new device model needed.

### Changes
- **omap3630.c**: `sysbus_create_simple("virtio-mmio", 0x4E000000, gpio_in(dev, 12))`
- **omap3630.h**: `OMAP3630_IRQ_VIRTIO0 12` (unused INTC line)
- **DTS**: `virtio_mmio@4e000000 { compatible = "virtio,mmio"; ... }`
- **Kernel**: `CONFIG_VIRTIO_MMIO=y`, `CONFIG_VIRTIO_BLK=y` (rebuilt zImage)
- **rootfs.ext4**: 64MB, created with `mkfs.ext4 -q -L rootfs -d initramfs rootfs.ext4`
  (populates from directory without sudo/mount)

### Bug: virtio-blk FLUSH Hangs
The `sync()` syscall hangs indefinitely. Reads and regular writes work fine —
only the REQ_PREFLUSH barrier command never completes.

**Root cause**: QEMU's block backend handles FLUSH asynchronously (coroutine
yields to event loop for host fsync()). The completion notification isn't
being delivered back to the guest — likely a TCG event loop scheduling issue
where the vCPU thread doesn't yield to the main loop to pump AIO completions.

**Diagnosis path**:
1. Initramfs boot + mount ext4 + write → works
2. Initramfs boot + mount ext4 + write + sync → hangs
3. Same with `nobarrier` mount option → sync works
4. Same with `cache.no-flush=on` → sync works
5. Confirmed: only the FLUSH/barrier I/O path is broken

**Workaround**: `cache.no-flush=on` on the QEMU `-drive` option. Tells the
backend to immediately report FLUSH as completed. Safe for emulation since
writes go directly to the host file.

### Validation Results
```

Root: /dev/root ext4
mount: /dev/root on / type ext4 (rw,relatime)
df: /dev/root 56.0M 1.7M 49.8M 3% /
```

Write test: `echo`, `dd`, `touch` all succeed. `sync` works with `cache.no-flush=on`.

### Boot Command (ext4 root)
```bash

  -kernel zImage -dtb omap3630-nokia-e7.dtb \
  -drive file=rootfs.ext4,if=none,format=raw,id=hd0,cache.no-flush=on \
  -device virtio-blk-device,drive=hd0 \
  -append "earlycon clk_ignore_unused root=/dev/vda rw init=/init" \
  -nographic
```
