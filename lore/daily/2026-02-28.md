## QEMU SYSSTATUS Acceleration — Boot 2.57s → 1.0s

### Problem
`omap_hwmod` softreset and `ti-sysc` OCP reset both poll SYSSTATUS registers
(bit 0 = RESETDONE) at various offsets. Our `create_unimplemented_device()`
stubs returned 0, causing 10ms timeout per device × 30+ devices = ~1.5s wasted.

### Fix
Replaced `create_unimplemented_device()` with custom `omap3_periph_stub` that
returns `0x00000001` for all reads. Created stubs for 30+ peripherals:
- GPIO1-6, WDT2, DSS, GPMC, MUSB
- AES1/2, RNG, SHAM, HDQ, McSPI1-4
- GPT3-11, GPT12, UART1/2/4
- HSMMC2/3, USB host, DSI1, SDMA

### Result
- Initramfs boot: **2.57s → 1.0s** (2.5x faster)
- Zero softreset timeouts (was 23 hwmod + 7 ti-sysc)
- Errors reduced from 21 to 7 (all expected: I2C not wired, MUSB not impl)

## Tasks 016+017: SDRC + Idle Handler — SOLVED

### Problem chain
1. Ext4 root boot hangs at "PM: genpd: Disabling unused power domains"
2. Kernel idle handler calls `sleep34xx.S` SRAM code that polls SDRC_DLLA_STATUS
3. SDRC stub returns 0 → infinite DLL lock poll (8.4M reads/20s measured)

### Fix 1: SDRC register model (QEMU, Task 016)
Replaced `create_unimplemented_device("omap3-sdrc")` with proper register model:
- Tracks DLLA_CTRL writes, defaults DLLA_CTRL=0x0C (DLL enabled + lock mode)
- Derives DLLA_STATUS from DLLA_CTRL (bit 2 = locked when bit 3 = enabled)
- `is_dll_in_lock_mode` check in sleep34xx.S:270 takes fast exit branch
- Agent (opus, worktree) found this approach

### Fix 2: CONFIG_CPU_IDLE=y (kernel, Task 017)
After SDRC fix, idle handler crashed at garbage PC address (c0a8d040 = BSS).
Root cause: `omap3_pm_idle()` calls `omap3_do_wfi()` directly, but that assembly
function ends with `ldmfd sp!, {r4-r11, pc}` — the matching `stmfd` is in
`omap34xx_cpu_suspend`, only called via cpuidle. Without CONFIG_CPU_IDLE, the
stack frame is mismatched → pops garbage into PC.

### Fix 3: vc.c NULL guard (kernel)
With cpuidle enabled, crash moved to `omap3_vc_set_pmic_signaling()` — NULL
deref of `vc.timings` because no TWL PMIC is configured. Kernel prints "No PMIC
info for vdd_core" but doesn't guard against it. Added:
```c
if (!vd || !c)
    return;
```

### Result
- **Ext4 root boot now works: ~0.8s to shell**
- Initramfs boot still works: ~1.0s
- No crashes in idle loop
- cpuidle governor "ladder" active

## Task 020: NLoader Complete RE — DONE

Extracted and analyzed NLoader bootloader (184KB ARM Thumb at rofs.img 0x144000).

### Method
- Extracted binary, mapped 174 format strings + 90 hardware strings
- Reconstructed complete boot sequence from debug printf messages
- Parsed flash TOC (32 partition entries)
- Attempted disassembly of key routines (Genio init, I2C init, charger detect)

### Key finding: GENIO_INIT is ENCRYPTED
The pad mux configuration table (most wanted data) is behind BB5 encryption
(entropy 7.99/8.0 = practically random). Cannot extract from firmware.

**However**: NLoader PRINTS every (address, value) pair from GENIO_INIT at boot.
Capturing UART output from real hardware = all pad mux data in plaintext.
**This makes UART access the #1 priority for real hardware work.**

### Other findings
- Flash partition table: 32 entries including SOS+CORE (Symbian), SOS+IVE3A/B
  (BCM2727 camera firmware, also encrypted), GENIO_INIT, KEYS, R&D flags
- NLoader does NOT configure SDRC — ROM bootloader does that
- Pearl-J = TWL5031 PMIC, VCore set via "Gazoo" regulator
- SmartReflex VCore calibration uses RAPU/YAMA efuse values per FOP
- Full MUSB OTG USB stack — "Nokia Mobile Device (NLoader mode)"
- Nokia uses HAL abstraction (not direct MMIO) — only 2 direct OMAP addresses found
- zlib 1.2.1.f-Nokia-mods-v2 decompression library
- Boot modes: Normal, Local (service), Test, Update, FOTA, Massmemory

## Session 4: TWL Validation + I2C Fix

### Task 012 Validation — DONE
Enabled CONFIG_TWL4030_CORE=y + REGULATOR_TWL4030=y + GPIO_TWL4030=y + TWL4030_POWER=y
in nokia_e7_defconfig. Rebuilt kernel (102-line defconfig, 3.8MB zImage).

**TWL5031 probes successfully in QEMU:**
```
twl 0-0048: PIH (irq 23) chaining IRQs 144..152
twl 0-0048: power (irq 149) chaining IRQs 152..159
twl4030_gpio twl4030-gpio: gpio (irq 144) chaining IRQs 160..177
vaux1: Bringing 0uV into 2800000-2800000uV
vaux2: Bringing 0uV into 2500000-2500000uV
```

### I2C Multi-byte Write Fix
**Bug**: `twl: Write failed (mod 10, reg 0x13 count 5)` — GPIO pullup 5-byte
bulk write timed out, adding ~1s to boot.
**Root cause**: After kernel acks XRDY in STAT (write-1-to-clear), QEMU never
re-asserted XRDY. Real HW re-asserts when FIFO has space and bytes remain.
**Fix**: In STAT write handler, re-set XRDY/RRDY if transfer still active.
**Impact**: Boot ~1.9s → ~0.94s. All TWL register writes succeed.

### Rootfs Rebuild
Ext4 rootfs was empty (just lost+found). Rebuilt from busybox _install with
`fakeroot mkfs.ext4 -F -L rootfs -d <dir>`. Added /init + /etc/init.d/rcS.
