# Code of Conduct

## The Short Version

Be excellent to each other. We're bringing Linux to a dead phone platform
using two irreplaceable devices and a lot of stubbornness. Anyone who wants
to help is welcome.

## The Slightly Longer Version

- **Be respectful** — this project involves hardware RE, security research,
  and working with legacy systems. People have different skill levels.
- **Share knowledge** — if you know something about Nokia hardware, OMAP3630,
  or Symbian internals, sharing it helps everyone.
- **Document dead ends** — failed approaches are just as valuable as successes.
  They save the next person from wasting time.
- **Don't distribute proprietary code** — discuss findings, document behavior,
  but don't redistribute Nokia firmware or copyrighted binaries.
- **Have fun** — we're hacking a 15-year-old phone. This is supposed to be enjoyable.

## Scope

This applies to all project spaces: GitHub issues, pull requests, wiki edits,
and any associated communication channels.

## Enforcement

If someone's being a jerk, open an issue or contact the maintainer directly.
Life's too short, and so is this phone's battery.
